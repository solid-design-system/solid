# Migration Guide: From `ui-header` to `sd-header`

The new `sd-header` is designed to replace the `ui-header`. Instead of mainly providing content via attributes, the `sd-header` component uses slots to allow for more flexibility and customization.

## ⚙️ Attributes

### ✨ New Attributes

#### [autoSpacing]

Setting the body spacing based on the height of the header.

<hr />

## ✍️ CSS Variables

### 🔀 Updated CSS variables

#### [--ui-header-content-width]

Renamed to `--sd-header-inner-width`.

#### [--ui-header-content-max-width]

Renamed to `--sd-header-inner-max-width`.

#### [--ui-header-content-padding-top]

Renamed to `--sd-header-padding-top`.

#### [--ui-header-content-padding-bottom]

Renamed to `--sd-header-padding-bottom`.

#### [--ui-header-content-padding-x]

Renamed to `--sd-header-padding-x`.
