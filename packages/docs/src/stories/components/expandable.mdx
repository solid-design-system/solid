    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Common Use Cases</h2>

    <div className="sd-prose">
      - Display non-essential information that users can choose to read if interested.
      - Prevent clutter by truncating long text blocks and allowing users to expand them as needed.
      - Implement in product descriptions to show more details on demand.
    </div>

    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Usage Guidelines</h2>

    <h3 className="sd-headline sd-headline--size-lg">Content and Labels</h3>

    <div className="sd-prose mb-6">
      - Use the ”default” slot to add content.
      - Use concise labels for the trigger (e.g., “Read more,” “Show more”).
      - The initial visible text should give a hint about the hidden content.
      - Ensure the content is supplementary and not critical.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Interaction and Usage </h3>

    <div className="sd-prose mb-6">
      - Avoid overloading with too much content. If a long text is required, consider other options for displaying it, e.g., triggering a sd-dialog.
      - Do not nest expandable sections within other expandable sections.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Expandable vs. Accordion</h3>

    <div className="sd-prose mb-6">
      - The use of sd-accordion is generally recommended over the use of sd-expandable: accordions give the user a quick scan header to decide if the content is relevant to them, offer a cleaner layout and are more accessible as a result.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Background</h3>

    <div className="sd-prose mb-6">
      - Use light background options like white, neutral-100, primary-100, or use a primary background when inverted.
    </div>

    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Accessibility Information</h2>

    <div className="sd-prose mb-6">
      - Expandables are intended to be used for text blocks, so no interactive elements would appear when expanding, thus the focus remains on the expandable trigger whether open or closed, adjusting the labelling for each state.
      - Use ARIA roles such as ”aria-expanded” to indicate the state of the expandable section. Ensure each section has a clear and descriptive label.
