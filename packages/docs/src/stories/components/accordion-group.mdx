    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Common Use Cases</h2>

    <div className="sd-prose">
      - Organise content into collapsible sections to save space.
      - Display FAQs where each question can be expanded to reveal its answer.
      - Structure lengthy content into manageable expandable sections. - Create navigational menus where each section can be expanded to show sub-items, mainly for small viewports.
    </div>

    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Usage Guidelines</h2>

    <h3 className="sd-headline sd-headline--size-lg">Slots</h3>

    <div className="sd-prose mb-6">
      - Use the ”summary” slot to add text to the header.
      - Use the ”default” slot to add content.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Content</h3>

    <div className="sd-prose mb-6">
      - Ensure the content is relevant and necessary; avoid including unrelated information.
      - Avoid using collapsible sections for content that needs to be always visible or is critical for immediate user attention.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Header Summary</h3>

    <div className="sd-prose mb-6">
      - Make sure the header summary clearly describes the content inside.
      - Keep header summaries concise to prevent them from wrapping onto multiple lines.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Behaviour</h3>

    <div className="sd-prose mb-6">
      - Keep only one section open at a time to prevent information overload, unless multiple open sections are necessary.
      - Avoid nesting collapsible sections within each other to prevent a confusing user experience.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Expandable vs. Accordion</h3>

    <div className="sd-prose mb-6">
      - The use of sd-accordion is generally recommended over the use of sd-expandable: the former gives the user a quick
      scan header to decide if the content is relevant to them, offers a cleaner layout and is more accessible as a result.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Background</h3>

    <div className="sd-prose mb-6">
      - Use light background options such as white, neutral-100, or primary-100.
    </div>

    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Accessibility Information</h2>

    <div className="sd-prose mb-6">
      - Use only for non-critical information. Hiding content can become a potential barrier, making it more challenging for
      users to access information.
      - Ensure that users can navigate through the accordion headers using the keyboard (e.g., Tab and Enter keys).
      - Use appropriate ARIA roles, such as ”aria-expanded” to indicate the state of the accordion
      sections or ”aria-live” regions to announce dynamic changes in the accordion content to screen reader users.
      - Use “close-others” attribute to keep only one item from the group open at a time, reducing the amount of information displayed at once and therefore reducing the cognitive load on the user.
      - Visit <sd-link href="https://www.figma.com/design/YDktJcseQIIQbsuCpoKS4V/Component-Docs?node-id=10708-5011&t=Z1FxSGWkmXG5r5sw-4" target="_blank">Solid DS Best Practices for WCAG Compliance</sd-link> to learn more about our accessibility standards.
