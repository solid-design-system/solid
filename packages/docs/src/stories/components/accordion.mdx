    <div className="sd-prose">

      - Interactive elements like a single FAQ item, where the user can expand to see the answer.
      - Useful in forms to hide optional sections that the user can expand if needed, keeping the form clean and concise.
    </div>

    <h2 className="sd-headline sd-headline--size-3xl mb-8 mt-12 text-black">Usage Guidelines</h2>

    <h3 className="sd-headline sd-headline--size-lg">Slots</h3>

    <div className="sd-prose mb-6">
      - Use the ”summary” slot to add text to the header.
      - Use the ”default” slot to add content.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Header Summary</h3>

    <div className="sd-prose mb-6">
      - Make sure the header summary clearly describes the content inside.
      - Keep header summaries concise to prevent them from wrapping onto multiple lines.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Content</h3>

    <div className="sd-prose mb-6">
      - Ensure the content is focused and necessary. If the information can be splitted into different meaningful units, consider using sd-accordion-group.
      - Avoid using collapsible sections for information that must always be visible.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Expandable vs. Accordion</h3>

    <div className="sd-prose mb-6">
      - The use of sd-accordion is generally recommended over the use of <sd-link href="https://solid-design-system.fe.union-investment.de/docs/?path=/docs/components-sd-expandable--docs">sd-expandable</sd-link>: the former gives the user a quick scan header to decide if the content is relevant to them, offers a cleaner layout and is more accessible as a result.
    </div>

    <h3 className="sd-headline sd-headline--size-lg">Background</h3>

    <div className="sd-prose mb-6">
